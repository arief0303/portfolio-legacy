Hide & Reveal Next Element

Sprint 2

To simplify the user interface of this site, i have made iteration to the 'Feedback' box to hide and reveal its content.

This is done by using JavaScript to target the next element(in the DOM) of the 'plus-minus' button using an if statement and changing the display: value in the CSS of the next element to either hide it(display: none) or to show it(display: block).


function toggleDocs(event) {
    if (event.target && event.target.className == 'plus-minus-toggle' || event.target && event.target.className == 'plus-minus-toggle collapsed') {

        var next = event.target.nextElementSibling;

        if (next.style.display == "none") {
            next.style.display = "block";

        } else {
            next.style.display = "none";
        }
    }
}

Responsive Web Design

Sprint 3

When viewing a website on a smaller screen real estate the website needs to scale accordingly otherwise the elements will clip.

In this sprint i have implemented a responsive design to this portofolio. Now it can be viewed properly on either big or small screen sizes.

Implementing this POC is relatively simple as it only took only took a few lines of CSS condition code to tell the browser to resize accordingly depending on the screen size.


@media only screen and (max-width: 40em) {
    .content {
        margin: 0px;
    }
}

Even though the syntax is simple, i needed to redesign the interface to fit accordingly so i have to make adjustment to the .content class to fit block-level elements.

Interactive Visual Art with ThreeJS

Sprint X

This was a pet project of mine and in Sprint X with the help of Mr. Chris Geene as my mentor, i have managed to finalize and polish this interactive visual art that i made using ThreeJS, a JavaScript 3D web graphics library.



Link to the interactive visual.

In this article i will describe in the detail of the steps i took to create this ThreeJS interactive visual.

1. Creating the scene

The way i created this interactive visual is first to define the basic components to create the scene, such as:

  • the scene itself
  • a camera to see the scene
  • a renderer to display the scene unto the canvas
// scene
var scene = new THREE.Scene();

// camera
var camera = new THREE.PerspectiveCamera(80, window.innerWidth / window.innerHeight, 0.1, 1000);

// renderer
var renderer = new THREE.WebGLRenderer({
antialias: true,
alpha: true
});
renderer.setSize(window.innerWidth, window.innerHeight); //set size of the renderer to window size
document.body.appendChild(renderer.domElement); //create canvas on the html body

2. Interactive Controls

Then i use an additional ThreeJS library module called OrbitControls.js to add interactive camera controls for the user to navigate around the 3D space by either using a mouse or touch control. Then i also modify a few parameters to add damping so that the camera will pan smoothly as the user navigates around the space.

// controls
var controls = new THREE.OrbitControls(camera, renderer.domElement);
controls.autoRotate = true;
controls.autoRotateSpeed = 1;
controls.enablePan = true;
controls.enableZoom = true;
controls.enableDamping = true;
controls.dampingFactor = 0.07;
controls.rotateSpeed = 0.8;

3. Objects

At this point, we would be having an empty scene that we can navigate around. So we can start filling it with objects.

To create an object we need three basic components, such as:

  • a geometry to define its shape
  • a texture
  • and a mesh to combine the texture and geometry

// pyramid object
var geometry1 = new THREE.TetrahedronGeometry(1.5, 0); //object1 geometry
var material1 = new THREE.MeshNormalMaterial(); //object1 material
//var mesh = new.THREE.Mesh(geometry1, material1); //Mesh should have been defined here but instead i put it in the cloner

4. Inheritance Cloner

I wanted to have a very dynamic visuals to fill the whole screen and i spent a lot of time experimenting on this. From my experience from using 3D software suites such as Cinema 4D and Blender, i know that objects can be 'cloned' using a cloner tool. So with the help of Mr.Chris Geene i was mentored on how to create an 'inheritance cloner' by creating multiple meshes(each time in slightly different location according to the index) of the object using for-loop statement then adding each meshes to an object array then to use an if statement to 'inherit' every newly created mesh to its predecessor thus mirroring the attributes(such as position and rotation) of the parent. 


// cloner
var objArray = [];
for (let i = 0; i <= 16; i++) {
var meshes = new THREE.Mesh(geometry1, material1);
meshes.position.set(i - 5, i, i);

if (i == 0) {
scene.add(meshes);
var primeMesh = meshes;
objArray.push(meshes);
} else {
primeMesh.add(meshes);
primeMesh = meshes;
objArray.push(meshes);
}
    }

5. Animating the objects to move based on the 'Golden Ratio'

The last step is to animate the objects, the goal i had in mind is to make the objects move(in rotation) with some certain flow. So i used Math.PI to make the objects move on the circumference of a circle according to the golden ratio.


function animate() {
    requestAnimationFrame(animate);
  
    controls.update();
    
    // golden ratio anim
    for (var x = 0; x < objArray.length; x++) {
      objArray[x].rotation.x += Math.PI / 2;
      objArray[x].rotation.y += Math.PI / 4;
      objArray[x].rotation.z += Math.PI / 4;
    }
  
    renderer.render(scene, camera);
};
